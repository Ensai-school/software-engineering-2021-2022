Patron composite

Le besoin

  • Représenter une structure avec une relation parent-enfant
  • Avec différents types (exemple branche/feuille)
  • Chaque type à une opération spécifique
  • Pour l'extérieur, il n'y a qu'un type visible

Exemple : représentation graphique

  • Formes géométriques complexes
  • Une forme peut-être divisée en sous forme
  • La forme la plus pure est le point

Patron observer

Le besoin

  • Notre système doit réaliser des actions dans certaines conditions
  • Les actions peuvent être identiques entre plusieurs conditions
  • On veut que le tout soit le plus indépendant possible
  • On n'a pas envie que les systèmes qui réaliser les actions filtrent les événements pour agir quand il faut

Exemple : station météo

  • Station météo qui enregistre différentes données
  • Quand les données changent mise à jour de différents affichages
